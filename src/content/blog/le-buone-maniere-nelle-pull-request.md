Le code review (o pull-request, ormai sono sinonimi) sono più una questione di comunicazione ed empatia che tecnica. O, come minimo, sono a pari livello.

Nelle code review i ruoli sono due:

- il submitter: chi richiede la code review;
- il reviewer: chi revisiona.

Entrambe le parti, in questo momento molto delicato quanto potente, hanno l'occasione di confrontarsi e la comunicazione (cosa scrivere e come scriverlo) è di fondamentale importanza.

Cosa dovrebbero fare, quindi, entrambi per estrarre il massimo da questa occasione?

Lo vediamo nel prossimo messaggio.
